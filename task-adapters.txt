## Task 1: Instagram browser adapter (replace private-api with Playwright)

Current file: packages/adapters/src/platforms/instagram.ts
Currently uses instagram-private-api which is fragile.

Replace publish() with Playwright browser automation:
- 1 account = 1 persistent Chrome profile (sessionDir from credentials, default: storage/sessions/instagram/<account.id>)
- headful by default (headless only if account.config.headless === true)
- Always start from https://www.instagram.com/
- Detect login wall: if login form visible, throw AdapterAuthError

Publish flows:

a) publishFeed(page, caption, mediaPaths[]):

  - Wait for preview, click Next

  - Click Share/Post button

b) publishStory(page, mediaPath):
  - From home, click Your Story circle or [aria-label*="story" i] in left sidebar/top
  - Or navigate to https://www.instagram.com/stories/create/
  - Upload image: input[type="file"] or file chooser trigger
  - Click Add to story / Share / Your story button

c) publishReel(page, caption, videoPath):
  - Click Create button
  - Click Reel option: [aria-label*="Reel" i] or text Reel
  - Upload video
  - Next
  - Caption
  - Share

  - if home page shows feed = healthy

## Task 3: Zalo - split into two adapters

Create two new files:

a) packages/adapters/src/platforms/zalo-bot.ts
   ZaloBotAdapter - using official Zalo OA API (not the KaiyoDev bot, but direct Zalo API)
   - testConnection(): call Zalo OA API with accessToken, check valid
   - publish(input): POST to Zalo OA Message API with text + optional image
   - crawl(): fetch recent messages via Zalo API
   - Credentials: { accessToken, oaId }
   - Use node-fetch or native fetch for HTTP calls
   - No browser/Playwright needed
   - Error: if 401/403 = AdapterAuthError, if 5xx = RetryableNetworkError

b) packages/adapters/src/platforms/zalo-web.ts
   ZaloWebAdapter - browser automation for personal Zalo account
   - Uses Playwright persistent context (same pattern as facebook/threads)
   - sessionDir: storage/sessions/zalo/<account.id>
   - headful by default
   - testConnection(): open chat.zalo.me, check login state
   - publish(input): send message to a target group/contact
     - navigate to target url or search for contact name
     - find message input, type text, send
     - if media: attach file
   - crawl(): scroll and read messages from a channel
   - Credentials: { sessionDir, targetUrl or targetName }
